# Privacy Policy

Last updated: 2026-03-14

This Privacy Policy explains how `repolyze` handles information when you use the open source project, its source code repository, and official release artifacts.

`repolyze` is designed as a local repository analysis tool. Based on the current project design and documentation, it analyzes already-cloned local Git repositories on your machine and generates statistics and reports locally.

## Scope

This policy applies to:

- the official `repolyze` source repository;
- official binaries and release artifacts published by the project maintainer; and
- documentation distributed with the project.

This policy does not automatically apply to forks, third-party builds, package mirrors, or modified versions of the software. Those distributions may follow different privacy practices.

## Information `repolyze` may process locally

When you run `repolyze`, the software may access and process information from the repositories and paths you choose to analyze, including:

- repository paths and directory names;
- Git history and commit metadata available in the local repository, such as author names, author email addresses, commit timestamps, and commit messages;
- repository structure and file system metadata, such as file names, file extensions, file counts, line counts, and byte counts;
- aggregated repository statistics, contribution summaries, and activity summaries;
- reports generated by you in JSON or Markdown format.

By design, this analysis is performed against local repositories on your device.

## What the project does not do by default

Based on the published project documentation and repository contents at the time of this policy:

- `repolyze` does not require a user account;
- `repolyze` does not operate a hosted analysis service;
- `repolyze` does not send repository contents or analysis results to the project maintainer by default;
- `repolyze` does not include built-in advertising or behavioral tracking;
- `repolyze` does not rely on remote Git hosting APIs to perform its core repository analysis.

If the project adds optional networked features in a future release, this policy should be updated before or when those features are released.

## How information is used

Information processed by `repolyze` is used to:

- analyze one or more local Git repositories selected by the user;
- calculate repository statistics and summaries;
- render interactive views in the terminal user interface;
- generate JSON or Markdown output requested by the user; and
- help users compare repositories locally.

## Where processing happens

The intended privacy model for `repolyze` is local processing on the user's machine.

That means repository analysis is expected to happen on your device rather than on project-controlled servers. Any reports written to disk are stored in the location you choose, or printed to standard output if you do not specify an output file.

## Sharing and disclosure

The project maintainer does not receive your repository data from the software by default.

Information may still be disclosed by you or your environment if you choose to:

- save reports to files and share them;
- paste command output into issue trackers, chats, or AI tools;
- commit generated reports into a repository;
- run the software on systems monitored by your employer, hosting provider, or device management tools.

Please review generated output before sharing it. Repository reports may reveal project names, directory structure, contributor identities, commit activity patterns, or other sensitive development metadata.

## Command-line and local environment considerations

Like many CLI tools, use of `repolyze` may expose limited operational data to your local environment, including:

- shell history containing commands you ran;
- terminal scrollback or log capture tools;
- operating system file access records;
- files you explicitly create with `--output` or similar workflows.

These behaviors are generally controlled by your shell, operating system, and development environment rather than by the project maintainer.

## Third-party services and distribution channels

The project may be distributed or hosted through third-party services, including:

- GitHub, for source hosting and release distribution;
- GitHub Actions, for release automation; and
- Homebrew, for package installation on macOS.

If you download the software, browse the repository, install via Homebrew, or interact with GitHub-hosted project resources, those third parties may collect information under their own privacy policies and terms. The `repolyze` project does not control those third-party practices.

## Data retention

Because `repolyze` is designed for local analysis, the project maintainer does not retain your repository analysis data by default through the software itself.

Retention of any generated data depends on your environment and choices, such as whether you:

- keep generated report files;
- preserve terminal logs or shell history;
- upload reports to external services; or
- use third-party package managers or hosting platforms.

You can generally delete local reports and related artifacts directly from your system when you no longer need them.

## Security

`repolyze` is intended to minimize privacy risk by operating on local repositories without requiring a hosted backend for core analysis.

However, no software environment is completely risk-free. You are responsible for:

- choosing which repositories to analyze;
- protecting access to your machine and working directories;
- storing generated reports securely; and
- reviewing outputs before sharing them externally.

If you use `repolyze` with repositories that contain confidential, proprietary, personal, or regulated information, evaluate that use under your own legal, contractual, and security obligations.

## Open source development and forks

Because `repolyze` is open source, anyone may inspect the source code and, subject to the license, create modified versions. Modified versions, forks, unofficial packages, or downstream distributions may behave differently from the official project and may introduce new data practices.

You should review the source, release notes, and privacy terms for any non-official distribution you use.

## Children's privacy

`repolyze` is a developer tool and is not directed to children.

This Privacy Policy may be updated as the project evolves. Material changes should be reflected in the repository so users can review the current version before using new features.
